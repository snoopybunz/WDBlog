    • Week 5

    LinkedIn Learning

    What I have learnt?

    • LOOPS:
    • start from index 0
    • structure: for(initialization; condition; iteration) { // loop body }
    • Eumarative For Loops: Used for iterating over the properties of an object or the elements of an array.
    • 'for...in' iterates over keys and 'for...of' iterates over values
    • eg: for (var key in object) { console.log(key, object[key]); }
    • While loops: Structure: while (condition) { // loop body }
    • Basic While loops: Used when the number of iterations is not known beforehand. var i = 0; while (i < 5) { console.log(i); i++; }
    • Do-While loop: Guarantees the loop body is executed at least once. var i = 0; do { console.log(i); i++; } while (i < 5);

    Week 6

    What I have learnt?

    • Logical Operators:
      • Logical AND (&&):
      • True if both expressions are true.
      • Example: (2 < 5) && (3 >= 2) returns true.
      • Logical OR (||):
      • True if at least one expression is true.
      • Example: (2 < 5) || (3 >= 2) returns true.
      • Logical NOT (!):
      • Inverts true or false values.
      • Example: !true returns false.
    • Comparison and Casting:
      • Comparison Operators:
      • Use == for value equality.
      • Use === for type and value equality.
      • Casting Best Practice:
      • Use == for value equality.
      • Use === for type and value equality.
    • Functions:
      • Functions Overview:
      • Reusable code blocks.
      • Eliminate redundant code.
      • Declaring Simple Function:
      • Syntax: function functionName() { // code }.
      • Example: function sayHello() { document.write('Hello'); }.
      • Function with Parameters:
      • Syntax: function functionName(param1, param2) { // code }.
      • Example: function getArea(width, height) { return width * height; }.
      • Calling Functions:
      • Use function name with parentheses.
      • Example: sayHello();, getArea(3, 5);.

      Week 12

      Setting Up RESTDb and Creating a Simple Contact Form

      What I have learnt?

      • Access to RESTDb and its sample code requires a CORS API key. The API key is crucial for accessing information through the RESTful API.
      • API key creation involves navigating to the "Settings" menu. Specific API key settings can be configured, including allowed methods and whitelisted URLs.
      • A CRUD system is being developed, focusing on Create, Read, Update, and excluding Delete for now. Bootstrap documentation is provided for creating a simple contact form. Code is available on Replit for forking.
      • Debugging tools like Chrome Inspectors can be utilized. Network calls provide information for debugging. Proper whitelisting of URLs in RESTDb is emphasized.
      • Inserting the API key in Postman for testing is highlighted. Universal access can be allowed during testing by setting the whitelist to "*". Testing involves setting the API key, ensuring proper whitelisting, and sending JSON data.

      Week 14

      Converting SVG to Lottie

      What I have learnt?

      • LottieFiles offers a tool for converting SVG to Lottie animations. Steps involve signing up, using the online tool, choosing animation type, and saving the result.
      • SVG files can be exported from software like Illustrator or obtained from online resources. Attribution is recommended for the use of external SVG resources.
      • Lottie Editor allows modification of animations, including changing dimensions and colors. The editor provides options to configure the number of frames per second, graphic dimensions, and animation length.
      • Limitations exist in converting only proper shapes to layers. For more precise animations, After Effects with the Bodymovin plugin is recommended.
